# Arcadia Repository Analysis

Arcadia is an AI-powered interactive web app built with React, TypeScript, and Google Gemini API.
It presents a secret-society/underground-auction narrative layered over a culinary discovery experience.

Repository: [joeljin20/Arcadia](https://github.com/joeljin20/Arcadia)

## Project Overview

Arcadia is a themed application styled around hidden societies and ritual access. The current implementation centers on recipe discovery, AI-generated initiation content, and encrypted auction-style artifacts.

## Tech Stack

- Frontend: React + TypeScript
- Build tool: Vite
- Styling: Tailwind CSS + custom CSS
- Animation: Motion (`motion/react`)
- Icons: Lucide React
- AI: Google Gemini API (`@google/genai`)
- Runtime/package manager: Node.js + npm

## Core Architecture (4-State App)

The app runs as a multi-state experience:

1. `ALCHEMY` state
- Recipe discovery dashboard
- Searchable recipe catalog (`PRD_RECIPES`)
- Category filtering + card grid + detail modal
- Secret phrase trigger (`et in arcadia ego`) to unlock initiation

2. `INITIATION` state
- Camera-based vision check via Gemini image analysis
- Initiation puzzle generated by Gemini with fallback puzzle
- Success transitions user into dashboard

3. `DASHBOARD` state
- Auction/syndicate-style view of encrypted lots
- Daily AI-generated prophecy flavor content
- Displays data from local mock storage

4. `ADMIN` state
- Create new lots with title, description, bid, image, and decryption key
- Applies cipher encoding before persistence
- Saves to localStorage-backed mock database

## Key Features

### Search and Discovery

- Full-text recipe search
- Category filtering
- Featured content and detail view interactions

### Security and Cipher Layer

- Character substitution cipher (`src/logic/cipher.ts`)
- Encoded lot fields with key-based reveal

### AI Integrations

Gemini is used for:
- Initiation puzzle generation
- Prophecy/lore generation
- Vision-based ritual verification

### Audio Layer

- Web Audio API synthesis in `src/services/audio.ts`
- Ritual feedback tones for state changes

### Persistence

- localStorage-backed service in `src/services/mockDB.ts`
- Stores auction lots and user flow state for demo continuity

## Run Locally

Prerequisite: Node.js

1. Install dependencies:

```bash
npm install
```

2. Set your Gemini key in `.env.local`:

```bash
GEMINI_API_KEY=your_key_here
```

3. Start the app:

```bash
npm run dev
```
